O que é?

HTML, que significa "Hypertext Markup Language" (Linguagem de Marcação de Hipertexto), é uma linguagem de marcação utilizada para criar a estrutura básica de páginas web. Desenvolvido pelo World Wide Web Consortium (W3C), HTML permite que os desenvolvedores web definam e estruturem o conteúdo de uma página, indicando diferentes tipos de elementos e suas relações.

Tags

As tags HTML são elementos fundamentais na construção de páginas web. Elas são usadas para estruturar e organizar o conteúdo de uma página, além de fornecer informações sobre o tipo de conteúdo presente. Cada tag HTML é envolvida por colchetes angulares (< e >), e muitas delas possuem uma tag de abertura e uma correspondente tag de fechamento, que é identificada pelo mesmo nome precedido por uma barra (). No entanto, algumas tags, como as de imagem, não possuem uma tag de fechamento.

Aqui estão algumas tags HTML comuns e suas funções:

  1. <!DOCTYPE html>: Define a versão do HTML que está sendo utilizada. Geralmente, é a primeira linha de um documento HTML.
  2. <html>: Define o início e o fim do documento HTML.
  3. <head>:Contém informações sobre o documento, como o título da página, meta informações, ligação com folhas de estilo (CSS), e scripts.
  4. <title>:Define o título da página, que é exibido na barra de título do navegador.
  5. <body>: Contém o conteúdo principal da página, como texto, imagens, links, formulários, etc.
  6. <h1,2,3,4,5 e 6>: São cabeçalhos de diferentes níveis, indicando a importância do texto. <h1>: é o mais importante e <h6>: é o menos. Eles são frequentemente usados para títulos e subtítulos
  7. <p>: Define um parágrafo de texto.
  8. <a>: Cria um link para outra página ou recurso. O atributo href especifica o destino do link.
  9. <img>: Insere uma imagem na página. O atributo src especifica o caminho da imagem.
  10. <ul>: Usadas para criar listas não ordenadas.
  11. <ol>: Usadas para criar listas ordenadas.
  12. <li>: É usado para cada item da lista.
  13. <div>: Define uma divisão ou seção genérica em um documento. É frequentemente usado com CSS para aplicar estilos ou para agrupar elementos relacionados.
  14. <span>: Semelhante ao <div>, mas é usado para aplicar estilos a partes específicas de texto.
  15. <form>: Define um formulário que pode conter elementos como campos de texto, botões, caixas de seleção, entre outros.
  16. <input>: Usado dentro de um formulário para criar campos interativos, como campos de texto, botões de rádio, checkboxes, etc.
  17. <br>: Cria uma quebra de linha.

Entendo comunicação Client x Server

A comunicação entre cliente (client) e servidor (server) em um contexto de desenvolvimento web é essencial para a interação dinâmica entre os usuários e os recursos armazenados no servidor. Aqui está uma visão geral básica de como a comunicação cliente-servidor funciona em um ambiente web:

  1. Requisição do Cliente (Client Request): O processo começa quando um cliente (geralmente um navegador da web) faz uma requisição para acessar uma página ou recurso específico. Isso é feito através de uma solicitação HTTP (Hypertext Transfer Protocol).
  2. Envio da Requisição ao Servidor: A requisição do cliente é enviada para o servidor web correspondente. A requisição contém informações como o método da requisição (GET, POST, etc.), o URL do recurso desejado e outros cabeçalhos HTTP.
  3. Processamento da Requisição no Servidor: O servidor recebe a requisição e executa o processamento necessário para atender à solicitação do cliente. Isso pode envolver acessar bancos de dados, executar lógica de negócios, ou recuperar arquivos estáticos.
  4. Geração da Resposta pelo Servidor:Com base na requisição do cliente, o servidor gera uma resposta. Essa resposta geralmente é composta por um código de status HTTP, cabeçalhos e o corpo da resposta, que contém os dados solicitados ou informações sobre o processamento da requisição.
  5. Envio da Resposta ao Cliente: A resposta gerada pelo servidor é enviada de volta ao cliente. Isso ocorre através do protocolo HTTP. O navegador do cliente interpreta a resposta e executa ações apropriadas, como exibir a página solicitada ou lidar com erros.
  6. Processamento do Lado do Cliente: No lado do cliente, o navegador interpreta o conteúdo da resposta e pode executar scripts JavaScript, renderizar HTML, processar folhas de estilo (CSS) e interagir com o usuário.
